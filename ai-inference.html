Two Worlds of AI Inference
Predictive AI
  • Classification, regression, object detection
  • Single forward pass — low latency (<10ms typical)
  • Compute-bound, CPU-friendly, batch-optimized

  • Autoregressive — token-by-token, high latency
  • Memory-bound, GPU-intensive, KV cache hungry
  • Rapidly evolving — vLLM, TensorRT-LLM, SGLang

The Core Challenge
Why LLM Inference Is Hard
Autoregressive Bottleneck
Each token depends on all previous tokens. You can't parallelize output generation — it's inherently sequential.

1 token = 1 full model forward pass
Memory-Bound Decode
Decode phase reads the entire model weights per token, but does minimal compute. Bottleneck is memory bandwidth, not FLOPS.

GPU utilization often < 5% during decode
KV Cache Growth
Attention key-value pairs cached for every token in the sequence. A 4K prompt on a 70B model needs ~1.34 GB of KV cache.

Scales linearly with sequence length × batch size
Prefill ≠ Decode
Prefill is compute-bound (process all input tokens at once). Decode is memory-bound (generate one token at a time).

Different bottlenecks → need different optimization

03 optimization10 / 18
Precision & Memory
Quantization & KV Cache
Weight Quantization
FP8 — The New Default
Production standard in vLLM since early 2026. FP8 KV cache reduces decode cost to 54% of BF16. Fixed earlier FA3 accuracy issues.
AWQ & GPTQ
Recommended INT4 methods. AMD Quark supports both + Rotation + SmoothQuant. Best when VRAM is tight.
Production Rule
Blackwell / Hopper → FP8
VRAM constrained → AWQ INT4
Edge / mobile → GGUF 4-bit
KV Cache Optimization
PagedAttention (vLLM)
Virtual memory for KV cache. Eliminates fragmentation, enables dynamic allocation. Near-zero waste on cache memory.
Prefix Caching (SGLang)
RadixAttention reuses KV cache across requests sharing the same prefix. Massive wins for system prompts, RAG, multi-turn.
The Scale Challenge
4K prompt on 70B model = ~1.34 GB KV cache. For 128K context, this grows to ~43 GB. Compression and offloading are active research areas.

Speculative Decoding
How It Works
Draft model
→
Generate K candidates
Target model
→
Verify all K in parallel
Accept
→
First N correct tokens
  • Output is mathematically identical to standard decoding
  • Speedup depends on acceptance rate — how well the draft model predicts the target
  • Best gains on structured / predictable text (code, math, templates)
Results (2025-2026)
DFlash on TPU v5p
3.13x
Average speedup. 6x on math/coding tasks. 2.29x end-to-end serving improvement.
Production Adoption
vLLM — Built-in speculative decoding support
SGLang — Native speculative with RadixAttention
TensorRT-LLM — Medusa and Eagle variants
When to use
Latency-sensitive apps where you can tolerate a small draft model. Not free — draft model uses GPU memory.

Architecture Shift
Disaggregated Prefill / Decode
The Idea
Prefill is compute-bound — process all input tokens at once.
Decode is memory-bound — generate one token at a time.

Solution: Run them on separate, optimized hardware pools.
Prefill nodes
→ KV →
Decode nodes
  • Each pool sized for its bottleneck
  • KV cache transferred between nodes (needs 4.5 GB/s+ bandwidth)
  • Production at Meta, LinkedIn, Mistral, HuggingFace
Key Players
NVIDIA Dynamo
Announced at GTC 2025. First-party framework for disaggregated LLM serving on NVIDIA GPUs.
llm-d (CNCF Sandbox)

Dollars & Sense
Cost Optimization Strategies
2-3x
Continuous Batching
Replace static batching with continuous (vLLM, TGI, TensorRT-LLM default). New requests join mid-batch. Single biggest throughput lever.
60-80%
Caching + Routing
Semantic caching for repeated queries. Smart routing to cheapest capable model. Prefix caching (SGLang) for shared system prompts.
5-10x
Batch Jobs
Non-realtime workloads on spot GPUs with large batch sizes. 5-10x cheaper than always-on serving for offline tasks.
Serverless / Scale-to-Zero
RunPod, Modal, Baseten — per-second billing with scale-to-zero. No idle GPU cost. Best for bursty workloads.
Spot Instances
60-90% cheaper than on-demand. Requires fault-tolerant serving with checkpointing. Works well with disaggregated arch.

Wrap-Up
Key Takeaways
01
Inference dominates AI compute (67%). Optimization has become as important as model training.
02
vLLM is the production default. SGLang wins on prefix-heavy workloads. TensorRT-LLM for max throughput on NVIDIA.
03
Disaggregated serving is the biggest architecture shift since continuous batching — separate prefill and decode for 2x+ throughput.
04
Custom silicon delivers 10x+ throughput (Cerebras, Groq) but with ecosystem lock-in. NVIDIA keeps 80% market share.
05
Edge inference is no longer aspirational — it's production. LLMs run on $35 Raspberry Pis, multimodal models hit sub-100ms on edge GPUs. OpenAI-compatible API means zero lock-in across 100+ providers.
